# Grill me — abd-ux-mockup

**Mechanics:** [`common/reference/grill-me-with-practice-skill.md`](../../../../common/reference/grill-me-with-practice-skill.md) — one question at a time; generate-to-learn when enough is shared.

Ask until the mockup at this fidelity level is unambiguous:

- Who uses each control and what decision are they making — or is the control type chosen without knowing the decision?
- What does the user see when things go wrong — empty lists, validation failures, permission denials, partial data?
- Which action matters most — and does control hierarchy (primary, secondary, tertiary) reflect that?
- Why this control type and not another — dropdown with two options, text field with 200 values?
- What happens immediately after the user acts — and is that feedback visible in the wireframe?
