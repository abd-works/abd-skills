# Grill me — abd-ux-information-architecture

**Mechanics:** [`common/reference/grill-me-with-practice-skill.md`](../../../../common/reference/grill-me-with-practice-skill.md) — one question at a time; generate-to-learn when enough is shared.

Ask until the IA at this fidelity level is unambiguous:

- Who actually uses each screen and what are they trying to accomplish — or is "the user" generic?
- Where does one screen end and another begin — feature split or user-perceived place?
- Does the site map match the user's mental model — linear flow, hub, or something else?
- What screens are missing — error recovery, first-time setup, empty states, admin overrides?
- Which content types belong on which screen — or is everything on an undifferentiated dashboard?
- What triggers each transition — deliberate choice, automatic redirect, or error?
