# Grill me — abd-architecture-blueprint

**Mechanics:** [`common/reference/grill-me-with-practice-skill.md`](../../../../../common/reference/grill-me-with-practice-skill.md) — one question at a time; generate-to-learn when enough is shared.

Ask until mechanisms, modules, and runtime flow are concrete:

- For each mechanism, can you describe the concrete code constraint it places — not just a label?
- Are modules drawn around business capabilities, or folder/framework conventions?
- Does the runtime flow describe actual behavior, or typical behavior for systems like this?
- Do any two mechanisms overlap, or is there a gap no mechanism owns?
- Do test tiers match how the team actually tests — which tier is least likely to be written?
