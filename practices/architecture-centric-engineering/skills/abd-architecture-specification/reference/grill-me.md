# Grill me — abd-architecture-specification

**Mechanics:** [`common/reference/grill-me-with-practice-skill.md`](../../../../../common/reference/grill-me-with-practice-skill.md) — one question at a time; generate-to-learn when enough is shared.

Ask until domain-to-file mapping and mechanism boundaries are concrete:

- When a new domain concept appears, is file placement and naming obvious — or a judgment call every time?
- For each mechanism, can you point to exactly where its code starts and stops?
- Does the walkthrough trace a real request path, or an idealized sequence?
- What unique architectural risk does each test tier prove?
- Where do stories not fit the pattern cleanly — and are those seams documented?
