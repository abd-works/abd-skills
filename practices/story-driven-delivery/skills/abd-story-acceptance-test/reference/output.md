# Output — abd-story-acceptance-test

This skill does **not** write under `docs/`. Tests follow the host project's conventions.

## Where to write (`<tests-folder>`)

1. **The path the user told you to use.**
2. **Where the project already keeps tests** — `tests/`, `test/`, `spec/`, or language-conventional test folder.
3. **The workspace root** or a sensible language-default location.

Do **not** invent a predetermined folder name.

## File names

Use the target language's test-discovery convention (`test_<scenario>.py`, `<Scenario>Test.java`, `*.test.ts`, etc.).
