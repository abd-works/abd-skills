# Built phase bundles (`content/parts/phases/built/`)

Each file is generated by **`scripts/build.py`** via **`MapsInstructions`** (same assembly as **`python scripts/generate_prompt.py --phase <slug> --mode dynamic`**). **`--mode static`** reads these files when present.

Legacy copy: **`content/built/phases/<slug>.md`** (same bytes).

Each bundle contains:

1. **Solution analyst role** — `content/parts/solution-analyst-role.md` (headings demoted).
2. **Rules** — `rules/*.md` filtered by YAML `phase_files` / `every_phase`.
3. **Library** — excerpts per **`PHASE_LIBRARY_SLICES`** in **`skill-config.json`**.
4. **Phase steps** — source `content/parts/phases/<slug>.md` only (no embedded role).
5. **Critical quality steps** — `content/parts/library/critical-quality-steps.md` plus optional focus lines from **`phase_critical_quality_notes`** in **`skill-config.json`**.
