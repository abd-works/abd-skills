# Skill Rule Workflow

> **Canonical file:** [`skill-workflow.md`](./skill-workflow.md) — this file exists so practice skills can keep linking `../common/skill-rule-workflow.md` after deploy.

Read **`skill-workflow.md`** when any practice skill tells you to. It defines output path resolution, read-gates, validate output (AI pass + scanner pass), diagram delegation, and the correction loop.

Practice families may add a **`skills/common/`** folder with extensions (deliverables conventions, diagram commands, shared validate items). When the active skill links there, read those files after `skill-workflow.md`. Story-driven delivery extensions deploy to **`common/story-driven-delivery/`**.
